# **2026中兴捧月赛题（初赛）**

## **赛题方向**

兴享智家

# **赛题背景**

在端侧AI与智能体技术快速发展的今天，如何在算力受限的环境中实现高效、智能、可落地的场景服务，成为产业界与学术界共同关注的焦点。在家庭终端领域，中兴通讯依托超1亿台年家庭终端发货量和对全球亿万家庭的服务基础，以家庭连接和AI的融合创新，提供AI家庭网算屏体“四大件”（[https://www.zte.com.cn/china/hottopics/ai_home.html](https://www.zte.com.cn/china/hottopics/ai_home.html)），持续推动家庭智能化升级，致力实现让AI服务于家庭每个成员。

当然，智能体的落地不仅限于家庭场景。在校园、企业等环境中，同样存在大量可通过轻量化、本地化AI提升效率与体验的创新机会。本次赛题鼓励参赛者跳出固定框架，结合端侧AI技术，设计并实现面向真实场景的智能体应用，体现“小模型、大智慧”的技术魅力。我们期待通过本次比赛，激发青年学子在端侧AI与智能体领域的创新潜能，推动轻量化、场景化、人性化的智能技术落地，共同构建“让AI服务于每个角落”的未来智能世界。

# **赛题简介**

参赛队伍需基于端侧算力约束（如嵌入式设备、开发板或轻量化AI加速硬件），设计并实现一个场景化智能体系统。该系统应具备以下能力：

* 场景理解与任务规划：能理解用户在特定场景（如家庭、校园、办公）下的自然语言指令，并将其转化为可执行的任务流程。
* 工具调用与资源整合：可调用预设或自设计的工具函数（如设备控制、信息查询、服务调用等），完成复杂任务。
* 多轮对话与状态保持：在对话中保持上下文连贯，支持多轮交互与任务延续。
* 高效推理与实时响应：在有限算力下实现低延迟、高并发的智能响应。

创新方向包括但不限于：

* 基于轻量模型的意图理解与槽位填充
* 端侧Agent架构设计与任务规划
* 多模态交互与场景感知
* 模型压缩、蒸馏与量化技术
* 本地知识库与检索增强生成
* 低资源环境下的持续学习与适应

为引导创新落地，提供以下三类场景（不强制限定，选手可自选其他合理场景）：

* 家庭场景（推荐）：智能家居控制、家庭健康管理、儿童教育陪伴、老人关怀提醒等。
* 校园场景：实验室设备管理、课程助手、校园导览、学习伙伴等。
* 办公/中小企业场景：会议调度、文档处理、访客接待、能耗管理等。

选手需在方案中明确所选场景，阐述其痛点与创新点，并展示智能体如何在该场景中提供价值。

# **已知条件**

（1）本赛题为团队赛，每个团队的参赛者至少1人、至多2人（必须为同一学校学生），一人最多只能属于一个团队，不同团队之间禁止任何形式的抄袭。

（2）赛事举办方采用多专家盲审方案对提交文件进行五个维度的评分，计算总得分。具体细则参见“评分规则”。

# **限制的提交语言****及语言版本**

* 开发环境：推荐使用常见嵌入式平台，也可使用模拟的轻量化AI环境。
* 算力约束：本地模型推理应在典型端侧设备（不超过6 TOPS算力，4GB内存）可承载范围内。
* 隐私与安全：方案应优先考虑数据本地处理，避免依赖云端敏感数据传输。
* 代码语言：C/C++/Java/Python。

# **赛题数据**

赛题不提供场景语料，参赛者可自行设计合成。

# **评分规则**

打分维度包括以下五大维度，每个维度满分均为20分，总分100分：

（1）创新：创意方案是否具有新颖性，是否超越现有功能

（2）价值：创意方案是否可行，且具有商业价值和应用前景

（3）成本：创意方案是否成本低，占用资源少且性能高

（4）演示：创意方案是否完成度高（优选逻辑：实机Demo > 仿真算法 > 方案idea）

（5）文档：创意方案设计文档是否质量高，完整详实且逻辑清晰

# **提交介绍**

参赛者在初赛结束前需提交包含以下文件的压缩包（ZIP格式），压缩包命名格式为“团队名-学校”，总大小不超过4G：

**1. 创新方案文档（PDF格式）：**

* 场景选择与痛点分析
* 系统架构设计
* AI模型与算法说明
* 工具函数设计与调用逻辑
* 性能优化措施
* 可扩展性与实用性评估

**2. 可运行代码：**

* 完整可部署的项目代码
* 代码说明文档
* 简易部署与测试脚本

**3. 演示视频（3分钟内）：**

* 展示智能体在模拟或真实环境中的运行效果
* 包括典型交互流程与多轮对话示例

文件目录结构如下：

团队名-学校

│   ├── algorithm

│   │   ├── 源代码

│   │   ├── 代码说明文档

│   │   ├── 演示视频

# **程序文件名和入口函数规定**

入口函数名为run，入口函数所在程序名为main

# **注意事项**

* 本赛题强调开放创新，不限制具体技术框架（如是否使用OpenClaw、LangChain等），鼓励选手灵活选用最新工具与模型。
* 允许使用公开预训练模型，但需说明优化与适配工作。
* 重点考察在算力约束下的实用性与创新结合能力，而非单纯追求模型参数量或准确率。
* 严禁抄袭和作弊，一经发现直接取消比赛资格。
