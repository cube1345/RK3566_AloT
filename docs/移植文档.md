# Agent ZX 跨平台移植文档

> 开发环境: Windows 11 + Git Bash | 目标平台: Raspberry Pi 5 (ARM64)
> 策略: 抽象隔离 + Mock 并行开发 + 配置驱动切换

---

## 一、移植策略总览

```
Windows 开发机                          Raspberry Pi 5 部署
┌─────────────────┐                    ┌─────────────────┐
│  Mock 传感器     │  ← 同一套代码 →   │  SGP30 / DHT11   │
│  (随机模拟数据)  │                    │  (真实 I2C/GPIO) │
│                 │                    │                  │
│  LLM CPU 推理    │  ← 同一套代码 →   │  LLM CPU 推理     │
│  (llama.cpp)     │                    │  (llama.cpp)     │
│                 │                    │                  │
│  设备 = 日志输出  │  ← 同一套代码 →   │  设备 = GPIO 控制  │
│  (无 GPIO)       │                    │  (gpiod / relay)  │
└─────────────────┘                    └─────────────────┘
```

**核心理念**: 一套代码, 通过环境变量切换运行模式, 零代码修改即可在 Windows 开发和 Pi 5 部署之间切换。

## 二、平台差异与隔离方案

### 2.1 Windows vs Linux (ARM64) 差异矩阵

| 差异项 | Windows 开发 | Pi 5 部署 | 隔离方式 |
|--------|------------|----------|---------|
| GPIO 控制 | 无 `/dev/gpiochip*` | gpiod (`/dev/gpiochip0`) | `try: import gpiod` + mock fallback |
| I2C 通信 | 无 `/dev/i2c-*` (smbus2 导入失败) | smbus2 (`/dev/i2c-1`) | `try: import smbus2` + mock fallback |
| UART 串口 | COM 口 | `/dev/ttyAMA0` | `try: import serial` + mock fallback |
| CPU 架构 | x86-64 | ARM64 (aarch64) | llama.cpp 分别编译 |
| Python 版本 | 3.13 (Conda) | 3.11+ (venv) | 标准库 API 兼容, 无差异 |
| 文件路径 | `C:\...` 反斜杠 | `/home/pi/...` 正斜杠 | `pathlib.Path` 统一处理 |
| 行尾符 | CRLF | LF | Git autocrlf 自动处理 |

### 2.2 硬件抽象层

```
sensors/base.py  ← BaseSensor (抽象接口)
    │
    ├── sensors/mock.py       ← MockCO₂, MockTempHumid, MockLight, MockMotion
    │                            MOCK_SENSORS=1 时加载, 生成模拟数据
    │
    ├── sensors/sgp30.py      ← SGP30Sensor (I2C, smbus2)
    ├── sensors/co2.py        ← CO2Sensor (UART, pyserial)
    ├── sensors/dht11.py      ← DHT11Sensor (GPIO, gpiod)
    ├── sensors/temp_humid.py ← TempHumidSensor (I2C, smbus2)
    ├── sensors/light.py      ← LightSensor (I2C, smbus2)
    └── sensors/motion.py     ← MotionSensor (GPIO, gpiod)
```

所有传感器实现 `BaseSensor` 接口, `SensorManager.register()` 统一注册, 上层代码不感知具体实现。

### 2.3 设备驱动隔离

```python
# devices/light.py — 模式隔离示例
class LightDevice(BaseDevice):
    def __init__(self):
        if _HAS_GPIO:
            self._init_gpio()     # Pi 5: 真实继电器控制
        # Windows: _req=None, control() 仅日志记录

    def control(self, action, **params):
        self._state = action      # 状态记录 (跨平台通用)
        self._apply()             # GPIO 写入 (Pi 5 生效, Windows 空操作)
        return f"灯光: {self._state}"  # 返回值 (跨平台通用)
```

所有设备在 GPIO 不可用时优雅降级: 状态仍记录, 日志仍输出, 返回值一致, 仅物理控制跳过。

## 三、配置驱动切换机制

### 3.1 环境变量一览

```bash
# 运行模式
AGENT_MOCK=1          # 1=Mock传感器, 0=真实传感器
AGENT_MOCK_WEATHER=1  # 1=Mock天气, 0=真实API

# 传感器选择
AGENT_SENSOR_CO2=sgp30    # sgp30(I2C) | mhz19b(UART)
AGENT_SENSOR_TEMP=dht11   # dht11(GPIO) | sht30(I2C)

# 平台选择
AGENT_PLATFORM=pi5        # pi5 | rk3566 | rk3506

# GPIO
AGENT_GPIO=1              # 1=启用真实GPIO, 0=禁用
AGENT_RELAY_LOW=1         # 1=低电平触发, 0=高电平触发

# 硬件参数
AGENT_I2C_BUS=1           # I2C 总线编号
AGENT_GPIO_CHIP=/dev/gpiochip0
AGENT_UART=/dev/ttyAMA0

# LLM
AGENT_LLM=models/Qwen2.5-1.5B-Instruct-Q4_K_M.gguf
```

### 3.2 加载流程

```
main.py
  → config.py (读取环境变量, 单次初始化)
    → MOCK_SENSORS = env("AGENT_MOCK")
    → SENSOR_CO2 = env("AGENT_SENSOR_CO2")
    → GPIO = {...} (按 PLATFORM 选引脚表)
    → I2C_ADDR = {...} (I2C 设备地址表)
    → DB_PATH = ROOT / "data" / "agent.db"
  → agent.py
    → _init_sensors():
        if MOCK_SENSORS → mock sensors
        else → 按 SENSOR_CO2/SENSOR_TEMP 选择真实驱动
  → 所有模块 import config, 无需重复读环境变量
```

### 3.3 一键切换命令

```bash
# Windows 开发 (Mock 模式)
set AGENT_MOCK=1
python main.py

# Pi 5 部署 (真实硬件)
export AGENT_MOCK=0 AGENT_GPIO=1 AGENT_SENSOR_CO2=sgp30 AGENT_SENSOR_TEMP=dht11
python main.py

# 或写入 .env 文件, Python 代码通过 os.getenv 读取
```

## 四、平台特定适配

### 4.1 GPIO 适配

```python
# config.py — 按平台选择引脚映射
if PLATFORM == "pi5":
    GPIO = {
        "relay_light": 17,     # BCM17 → 物理 Pin11
        "relay_fan": 27,       # BCM27 → 物理 Pin13
        "ir_led": 22,          # BCM22 → 物理 Pin15
        "relay_purifier": 10,  # BCM10 → 物理 Pin19
        "dht11": 4,            # BCM4  → 物理 Pin7
        ...
    }
elif PLATFORM == "rk3566":
    GPIO = {  # 完全不同的引脚映射
        "relay_light": 11,
        "relay_fan": 13,
        ...
    }
```

同一套代码适配 Pi 5 和 RK3566 (不同 GPIO bank 编号体系)。

### 4.2 GPIO 库版本兼容

```python
# 兼容 gpiod v1 (旧) 和 v2 (新)
try:
    import gpiod
    _HAS_GPIO = True
    _GPIOD_V2 = hasattr(gpiod, "LineSettings")  # v2 标志
except ImportError:
    _HAS_GPIO = False
    _GPIOD_V2 = False

# v2 API: chip.request_lines(config={...})
# v1 API: chip.get_line(pin).request(type=...)
# 三目判断: if self._req (v2) else self._line (v1)
```

### 4.3 I2C 适配

```python
# 使用 smbus2 的 i2c_msg (非 SMBus 协议设备)
# SGP30 和 SHT30 都是纯 I2C, 不是 SMBus 寄存器模型
# 需用 i2c_rdwr() 而非 read_byte_data()

write_msg = smbus2.i2c_msg.write(addr, cmd_bytes)
bus.i2c_rdwr(write_msg)
time.sleep(0.012)
read_msg = smbus2.i2c_msg.read(addr, 6)
bus.i2c_rdwr(read_msg)
```

### 4.4 DHT11 GPIO 时序适配

```python
# DHT11 单总线协议 — 微秒级时序 (Python 不保证精度)
# 解决方案:
#   1. 忙等待 (perf_counter_ns) — 精度 ~1µs, 但受 GIL 影响
#   2. 数据完整性校验 (CRC-8)
#   3. 读取失败 → 返回缓存值
#   4. 最小读取间隔 1.5s (DHT11 硬件限制)
#
# Pi 5 实测: 成功率 ~85-95%, 偶发超时被缓存容错覆盖
```

## 五、依赖管理

### 5.1 依赖矩阵

| 包 | Windows | Pi 5 ARM64 | 备注 |
|----|---------|-----------|------|
| flask | pip | pip | 纯 Python, 跨平台 |
| smbus2 | pip (导入报错) | pip (正常) | Linux I2C 专用 |
| pyserial | pip | pip | 跨平台 |
| gpiod | pip (导入报错) | pip (正常) | Linux GPIO 专用 |
| pytest | pip | pip | 跨平台 |
| llama-cpp-python | pip (CPU) | 需源码编译 | C++ 编译依赖 |
| requests | pip | pip | 跨平台 |

### 5.2 安装命令差异

```bash
# Windows
pip install flask pyserial smbus2 gpiod pytest requests
pip install llama-cpp-python  # 预编译 wheel

# Pi 5 (额外步骤)
sudo apt install -y i2c-tools gpiod  # 系统级依赖
sudo raspi-config nonint do_i2c 0    # 启用 I2C
cd ~/llama.cpp && make -j4            # 编译 llama.cpp
pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cpu
```

## 七、移植检查清单

部署前逐项确认:

- [ ] Pi 5 系统更新 (`apt update && apt upgrade`)
- [ ] I2C 已启用 (`raspi-config`)
- [ ] `i2cdetect -y 1` 能看到 SGP30(0x58)+BH1750(0x23)
- [ ] gpiod 可用 (`python -c "import gpiod; print(gpiod.__version__)"`)
- [ ] llama.cpp 已编译 (`ls ~/llama.cpp/main`)
- [ ] 模型文件已就位 (`ls -lh models/*.gguf`)
- [ ] `.env` 配置为 `AGENT_MOCK=0`
- [ ] 继电器 LOW 触发极性与模块匹配 (`RELAY_ACTIVE_LOW`)
- [ ] 虚拟环境已创建+依赖已安装
- [ ] 测试通过 (`pytest -k "not TestSensorDrivers"`)
- [ ] Dashboard 可访问 (`http://<IP>:5000`)
